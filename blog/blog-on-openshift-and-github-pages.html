Blog on Openshift and Github Pages

五 23 一月 2015
By mirsking

考试终于结束了，复习这两周好辛苦的说：又是感冒，又是扭到脖子...

上次复习过程中将博客迁移到了Github Pages中，暂时分配了域名mirsking.tk，然后一直想添加一个二级域名feed.mirsking.tk作为RSS的域名，以便和原来的博客无缝转移。可惜Github Pages貌似只支持一个域名，无法 做二级域名转发，而且不支持.htaccess...

无奈VPS太贵，踌躇间，突然想起以前申请的免费云主机Openshift(Openshift的优点是长期免费，而且最重要的式支持git!!!)于是乎，折腾就此开始。现在记录下折腾的过程和成果。

折腾效果

博客域名:

 mirsking.tk CNAME到 mirsking.github.io //由于tk域名对国内搜索引擎不友好,所以这个域名只做github的备份用.

 mirsking.com CNAME到 domain-mirsking.rhcloud.com //这个作为主域名,兼容原来博客基本上所有的链接地址(原博客的部分链接做了删除...)
 feed.mirsking.com CNMAE到 feed-mirsking.rhcloud.com //这个作为feed的主域名

 mirsking.ml CNAME到 mirsking.com
 feed.mirsking.ml CNAME到 feed.mirsking.com

 wangpeng.ml CNAME到 mirsking.com
 feed.wangpeng.ml CNAME到 feed.mirsking.com

额, 不要骂我搞了这么多域名, ml/tk都是免费域名,只是为了防止被别人抢注, wangpeng.ml本来是被别人占着的,好不容易刚刚抢注到...

折腾过程

  1. Github Pages存储的是pelican生成的静态网页,具体看我上一篇博客.

  2. Openshift 上创建了两个app,一个用来托管mirsking.com主站,一个用来托管feed.mirsking.com. 没办法,谁让Openshift也不支持二级域名重定向呢. 不过不知道是否支持.htaccess, 等有空了再试验一下.

Openshift创建app并配置nginx

  • 登陆openshift,在Applications选项卡中添加Application

  • 选择最右下脚的Do it yourself(注意这里一定要选择diy,否则运行nginx就要停用已经跑起来的占用了端口服务)

  • cd到Openshift的TMP目录下载安装nginx,这里初次用Openshift的我感觉到了深深的恶意,没有找到Openshift的文档,不清楚Openshift到底又多少各变量,在此总结下:

Openshit变量总结

$OPENSHIFT_TMP_DIR          /tmp
$OPENSHIFT_DATA_DIR         ~/app-root/data
$OPENSHIFT_DIY_IP           your ip
$OPENSHIFT_DIY_PORT         your port

nginx安装过程

cd $OPENSHIFT_TMP_DIR
wget http://nginx.org/download/nginx-1.7.9.tar.gz
wget ftp://ftp.csx.cam.ac.uk/pub/software/programming/pcre/pcre-8.36.tar.gz

tar xzf nginx-1.7.9.tar.gz
tar xzf pcre-8.36.tar.gz

cd nginx-1.7.9
mkdir -p $OPENSHIFT_DATA_DIR/nginx
./configure --prefix=$OPENSHIFT_DATA_DIR/nginx --with-pcre=$OPENSHIFT_TMP_DIR/pcre-8.36
make 
make install

经过上述步骤nginx已经安装成功了,下边就是配置nginx并启动了,先:echo $OPENSHIFT_DIY_IP echo $OPENSHIFT_DIY_PORT, 获取application的ip和端口.

vim $OPENSHIFT_DATA_DIR/nginx/conf/nginx.conf`

找到

server{
    listen [ip:port];
    server_name [ip];
    location /{
        root your_site_dir
        index index.html
        }
}

修改为上边echo出来的ip和端口. root后边写自己的博客地址.

最后到$OPENSHIFT_DATA_DIR/nginx/sbin/目录运行nginx,nginx就成功跑起来了.

这里如果出现"Permission Denied"之类的错误,基本都是ip和端口没配置好,Openshift没给那些端口权限导致的.

好了,这次总结到此为止,开始写老板的大作业喽!!!
